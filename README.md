# Nameh.me — Self-Hosted Mail Platform

A production-grade, Docker-based email infrastructure designed to scale to 1 million+ users.

## Quick Start

```bash
# Clone the repository
git clone <repo-url> nameh && cd nameh

# Run automated setup
bash setup.sh

# Or manually:
cp .env.example .env       # Edit passwords
docker compose up -d       # Start everything
```

**That's it.** All 12 services boot, connect, and become healthy automatically.

## Service Map

| Service | What It Does | Local URL |
|---------|-------------|-----------|
| **Stalwart** | Core mail engine (SMTP/IMAP/JMAP) | `http://localhost:8080` |
| **Backend API** | Custom application layer (FastAPI) | `http://localhost:8000/api/docs` |
| **Roundcube** | Reference webmail frontend | `http://localhost:8888` |
| **Traefik** | Edge proxy + auto TLS | `http://localhost:8090` |
| **MinIO** | S3-compatible blob storage | `http://localhost:9001` |
| **Rspamd** | Spam filtering dashboard | `http://localhost:11334` |
| **PostgreSQL** | Primary database | `localhost:5432` |
| **Redis** | Cache & session store | `localhost:6379` |
| **ClamAV** | Antivirus scanning | Internal |

## First Steps After Boot

1. **Get Stalwart admin password:**
   ```bash
   docker logs nameh-stalwart 2>&1 | grep -i password
   ```

2. **Open Stalwart Admin UI** at `http://localhost:8080`
   - Settings > Server > Network > Set hostname
   - Management > Directory > Domains > Add your domain
   - Management > Directory > Accounts > Create a test user

3. **Test webmail** at `http://localhost:8888` (Roundcube)

4. **Explore the API** at `http://localhost:8000/api/docs`

## Email Client Setup

| Setting | Value |
|---------|-------|
| IMAP Server | `localhost` |
| IMAP Port | `993` (TLS) or `143` (STARTTLS) |
| SMTP Server | `localhost` |
| SMTP Port | `465` (TLS) or `587` (STARTTLS) |
| Username | `user@yourdomain` |

## Architecture

```
Internet -> Traefik (TLS) -> Stalwart (Mail)
                           -> Backend API (App Logic)
                           -> Roundcube (Webmail)

Stalwart -> PostgreSQL (metadata)
         -> MinIO/S3 (email blobs)
         -> Redis (lookup cache)
         -> Rspamd (spam) -> ClamAV (virus)
```

## Environment

All configuration is in `.env`. Copy from `.env.example` and customize:

```bash
cp .env.example .env
```

Key variables:
- `POSTGRES_PASSWORD` — Database password
- `MINIO_ROOT_PASSWORD` — Object storage password
- `SECRET_KEY` — Backend API signing key
- `DOMAIN` — Your domain (e.g., `nameh.me`)

## Production Deployment (nameh.me)

When ready for production:

1. **Update `.env`:**
   ```
   ENVIRONMENT=production
   DOMAIN=nameh.me
   MAIL_HOSTNAME=mail.nameh.me
   ADMIN_DOMAIN=admin.nameh.me
   WEBMAIL_DOMAIN=webmail.nameh.me
   API_DOMAIN=api.nameh.me
   ACME_EMAIL=admin@nameh.me
   ```

2. **Update Stalwart config** (`infrastructure/stalwart/config.toml`):
   - Change `hostname` to `mail.nameh.me`
   - Update database passwords to match `.env`

3. **DNS Records** (at your registrar):
   ```
   A     nameh.me          -> YOUR_SERVER_IP
   A     mail.nameh.me     -> YOUR_SERVER_IP
   MX    nameh.me          -> mail.nameh.me (priority 10)
   TXT   nameh.me          -> "v=spf1 mx -all"
   TXT   _dmarc.nameh.me   -> "v=DMARC1; p=quarantine; rua=mailto:dmarc@nameh.me"
   ```
   DKIM records are generated by Stalwart — check the admin UI.

4. **Remove override** (disables direct port exposure):
   ```bash
   mv docker-compose.override.yml docker-compose.override.yml.bak
   ```

5. **Enable HTTPS in Traefik** — already configured with Let's Encrypt.
